# CI Maturity and Telemetry Baseline

Canonical source: [`docs/CI_TELEMETRY_BASELINE.md`](../CI_TELEMETRY_BASELINE.md).

This chapter describes seLe4n's CI observability infrastructure, established by WS-B10.

## Purpose

CI telemetry provides timing and stability signals for every validation lane, enabling early detection of performance regressions and flaky tests without introducing an external telemetry backend.

## Data sources

Telemetry is generated by two repository-owned scripts:

| Script | Function |
|--------|----------|
| `scripts/ci_capture_timing.sh` | Wraps CI commands; records duration + exit code as JSON lines |
| `scripts/ci_flake_probe.sh` | Repeats selected lanes; records attempt-level outcomes for flake signal |

Artifacts are uploaded from three CI workflows:

1. **Lean CI** (`.github/workflows/lean_action_ci.yml`) — docs, fast, smoke, full lanes
2. **Nightly** (`.github/workflows/nightly_determinism.yml`) — nightly + repeated smoke flake probe
3. **ARM64 fast** (`.github/workflows/platform_security_baseline.yml`) — platform security gate

## Telemetry artifact schema

### `timing.jsonl`

Each line records one command execution:

```json
{"lane": "smoke", "command": "./scripts/test_smoke.sh",
 "start": "2026-02-18T06:01:10Z", "duration_seconds": 112, "exit_code": 0}
```

### `flake_probe.jsonl`

Each line records one probe attempt:

```json
{"probe_id": "nightly-smoke-repeat", "attempt": 2,
 "command": "./scripts/test_smoke.sh", "duration_seconds": 114, "exit_code": 0}
```

### `flake_summary.txt`

Human-readable summary, e.g.: `nightly-smoke-repeat: stable (3 attempts, exit=0)`.

## Interpretation policy

| Signal | Threshold |
|--------|-----------|
| **Timing regression** | p95 duration grows >25% week-over-week for two consecutive snapshots |
| **Flake signal** | Repeated attempts yield mixed exit codes |

Flake signals are triaged as reliability incidents. Timing regressions are reviewed weekly.

## What WS-B10 delivered

- Command-level CI timing capture across all lanes
- Repeated-lane flake probe with configurable attempt count
- Telemetry artifact uploads to `.ci-artifacts/telemetry/`
- Weekly Lean toolchain drift proposals via Dependabot
- Explicit CodeQL policy rationale (informational/non-blocking) in `docs/CI_POLICY.md`
