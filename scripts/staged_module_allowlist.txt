# scripts/staged_module_allowlist.txt
#
# Per WS-RC R12.B: explicit allowlist of modules that are intentionally
# in the Platform.Staged transitive closure but NOT reachable from the
# production library entry point (SeLe4n.lean).
#
# The companion gate scripts/check_production_staging_partition.sh
# verifies that the live staged_only set equals this allowlist exactly.
# A module that enters production (becomes reachable from SeLe4n.lean)
# without being removed from this allowlist will fail the gate; a new
# staged module added without an allowlist entry will also fail.
#
# Format: <module>  # <category>: <one-line rationale>
# Categories:
#   marker              — file carries a "STATUS: staged for ..." marker
#   transitively-staged — pulled in by a marker-bearing module
#   infra               — platform infrastructure (bridges, anchors, helpers)
#   anchor              — the staging anchor file itself
#
# Last verified: WS-SM SM3.A landing.  Two modules moved OUT of the
# allowlist at SM3.A (both transitioned from staged-only to
# production-reachable):
#
#   * `Kernel.Concurrency.Locks.RwLock` — entered the production
#     import closure via `Model.Object.Types`'s new SM3.A.1 import.
#     Every kernel-object struct now carries a `lock : RwLockState`
#     field; the abstract operational spec is consumed structurally.
#   * `Kernel.Concurrency.MemoryModel` — entered production
#     transitively (via `Locks.RwLock` which imports it).
#
# The "STATUS: staged" markers in `Locks/RwLock.lean` and
# `Concurrency/MemoryModel.lean` were replaced with WS-SM SM3.A
# landing notes in the same cut per the implement-the-improvement
# rule.
#
# `Kernel.Concurrency.Locks.RwLockRefinement` was added to the
# allowlist by the WS-SM SM2.C landing (entry below; entry text
# preserved verbatim from that landing's WS-SM SM2.C.20 rationale
# at v0.31.9).  SM3.A does **not** remove this entry — the import
# direction is `RwLockRefinement → Locks.RwLock` rather than the
# reverse, so `Locks.RwLock`'s promotion to production does NOT
# transitively pull in `RwLockRefinement`.  A future phase
# (SM3.B `LockId.lookup` discharge, SM3.C `withLockSet` 2PL
# proofs) may consume the refinement bridge and promote it; until
# then it stays staged-only.
